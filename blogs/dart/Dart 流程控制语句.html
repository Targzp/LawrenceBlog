    Dart 流程控制语句

    Lawrence 2023/5/17

    # Dart 流程语句

    你可以使用下面的语句来控制 Dart 代码的执行流程：

    • if 和 else
    • for 循环
    • while 和 do-while 循环
    • break 和 continue
    • switch 和 case
    • assert

    # if 和 else

    Dart 支持 if - else 语句，其中 else 是可选的，比如下面的例子。

    if (isRaining()) {
  you.bringRainCoat();
} else if (isSnowing()) {
  you.wearJacket();
} else {
  car.putTopDown();
}

    # For 循环

    你可以使用标准的 for 循环进行迭代。例如：

    var message = StringBuffer('Dart is fun');
for (var i = 0; i < 5; i++) {
    message.write('!')
}

    var callbacks = [];
for (var i = 0; i < 2; i++) {
    callbacks.add(() => print(i));

    # Break 和 Continue

    使用 break 可以中断循环：

    # Switch 和 Case

    Switch 语句在 Dart 中使用 == 来比较整数、字符串或编译时常量，比较的两个对象必须是同一个类型且不能是子类并且重写 == 操作符。枚举类型非常适合在 Switch 语句中使用。

    每一个非空的 case 子句都必须有一个 break 语句，也可以通过 continue、throw 或者 return 来结束非空 case 语句。 不匹配任何 case 语句的情况下，会执行 default 子句中的代码：

    var command = 'OPEN';
switch (command) {
  case 'CLOSED':
    executeClosed();
    break;
  case 'PENDING':
    executePending();
    break;
  case 'APPROVED':
    executeApproved();
    break;
  case 'DENIED':
    executeDenied();
    break;
  case 'OPEN':
    executeOpen();
    break;
  default:
    executeUnknown();
}

switch (command) {
    case 'CLOSED':
        executeClosed();
        continue nowClosed;
        // 继续执行标签 nowClosed 后面的语句
    nowClosed:
    case 'NOW_CLOSED':
        // 在 CLOSED 和 NOW_CLOSED case 下都会执行的语句
        executeNowClosed();
        break;

    # 断言

    在开发过程中，可以在条件表达式为 false 时使用 —— assert(条件、可选信息); 语句来打断代码的执行。你可以在本文中找到大量使用 assert 的例子。下面是相关示例：

    // Make sure the variable has a non-null value.
assert(text != null);

// Make sure the value is less than 100.
assert(number < 100);

// Make sure this is an https URL.
assert(urlString.startsWith('https'));

    assert 的第二个参数可以为其添加一个 字符串消息。

    assert(urlString.startsWith('https'),
    'URL ($urlString) should start with "https".');
    1
    2

    assert 的第一个参数可以是值为布尔值的任何表达式。如果表达式的值为 true，则断言成功，继续执行。如果表达式的值为 false，则断言失败，抛出一个 AssertionError 异常。

    在生产环境代码中，断言会被忽略，与此同时传入 assert 的参数不被判断。
